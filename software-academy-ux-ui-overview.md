# Software Academy — UX/UI Overview

Сводка всех решений по дизайну. Документ-handoff для перехода к разработке (React 18 / TypeScript / Vite).

---

## 1. Что это и зачем

Корпоративный учебный портал для пользователей Manufacturing Platform: короткие, задачные видеоуроки (3–5 мин). Цели — сократить онбординг, снизить число обращений в поддержку, давать непрерывное обучение командам на производстве. Standalone-приложение со своей авторизацией и брендом. **Mobile-first**: основной сценарий — телефон в портретной ориентации.

Главная цель опыта: пользователь находит нужный ответ за пару минут и возвращается к работе. Academy должна ощущаться как продолжение промышленного продукта, а не как «сайт с видео».

---

## 2. Действующие лица и модель доступа

- **Разработчик / вендор** (вы): ведёт код, загружает видео, проставляет видимость по ролям, раскладывает уроки по категориям. Контент — **единая библиотека для всех клиентов**.
- **Администратор завода** (клиент, купивший продукт): получает логин+пароль/ссылку при покупке. Управляет **только людьми** — создаёт пользователей, назначает роли, активирует/деактивирует, сбрасывает пароли. Контент не трогает.
- **Работники**: учатся по своей роли.

Секретность обеспечивается тем, что доступ есть только у купивших продукт (единый контент, не пер-тенант).

**Роли и доступ к урокам:**
- Привилегированные — `Administrator`, `Manager`, `Site Manager`: видят **все** уроки + раздел `Users`. Пока равны между собой (ограничения для Manager/Site — на будущее).
- Рабочие — `Inspector`, `Welder`, `Fitter`: видят только уроки, помеченные под их роль.
- Модель расширяема: новые роли добавляются без переделки. Видимость = (роль пользователя ∈ роли урока) OR (роль привилегированная).
- Смена роли мгновенно перестраивает каталог пользователя. Ручного назначения уроков нет.

**Прогресс строго личный** — чужой прогресс не видит никто. Это убрало командную аналитику из системы.

Первый вход работника: администратор выдаёт логин + **временный пароль**, который работник меняет на постоянный при первом входе.

## 3. Информационная архитектура

**Слои:** Витрина (открыта всем, маркетинг — может жить на сайте компании) → Вход (логин+пароль) → Приложение.

**Навигация приложения** (адаптивная по роли):

| Раздел | Назначение | Кому виден |
|---|---|---|
| `Dashboard` | Личная домашняя (адаптивная) | Все |
| `Lessons` | Каталог/витрина уроков | Все |
| `Reports` | Личная статистика обучения | Все |
| `Users` | Управление людьми | Привилегированные роли |
| `Settings` | Язык, профиль, уведомления, субтитры, скорость, тема | Все |

- На широком экране — боковое меню; на телефоне — нижняя панель вкладок.
- Каркас навигации фиксирован в коде. **Категории уроков — динамические (CRUD на стороне вендора)**, т.к. отражают растущую платформу.

---

## 4. Разделы — что внутри

**Dashboard** — два состояния одного экрана:
- Новичок (нет прогресса): приветствие + список категорий с подсказкой «Start here».
- Вернувшийся: общий прогресс полосой в шапке + Continue learning + Recommended + Recently watched + Latest Academy news.

**Lessons** — каталог: категория → подкатегория → страница с шапкой (хлебные крошки + название + счётчик) и списком уроков. Отфильтрован под роль. Без персональных блоков. Может показываться витриной с горизонтальными рядами по категориям.

**Урок (плеер):**
- Vimeo Player сверху (встроенный 16:9 в портрете; полный экран в ландшафте — штатное поведение Vimeo).
- Структура «In this lesson» как главы с таймкодами: **Problem → Screen demonstration → Short explanation → Expected result**.
- Транскрипт (сворачиваемый), карточка «Up next».
- Прогресс автосохраняется: watch %, last position, started_at, completed_at. Кнопки «сохранить» нет.

**Reports** — личная статистика (объединили бывшие Progress и Reports): сводка сверху (общий %, завершено, самые просматриваемые), ниже список уроков со статусами.

**Users** (привилегированные роли) — 5 действий: создать пользователя (имя, роль, логин → временный пароль), просмотреть, сменить роль, активировать/деактивировать, сбросить пароль. Деактивация вместо удаления — прогресс сохраняется.

---

## 5. Основные сценарии (User Flow)

**Администратор:** покупка → доступ → `Users` → создать работника (имя, роль, логин) → система выдаёт временный пароль → передать работнику офлайн → работник входит, меняет пароль → видит каталог по роли.

**Работник (сварщик):** вход → `Dashboard` (continue/recommended — нужный урок в 1 клик) или `Lessons` (каталог по роли) → урок открыт → просмотр с автосохранением % и позиции → завершено → «Next lesson» или назад к работе.

## 6. Дизайн-система

**Палитра — графит + один акцент (бирюза):**
- Графит: Ink `#1C2024`, 800 `#2C313A`, 600 `#4A515B`, 400 `#7C838D`, 200 `#C9CDD3`, 100 `#E9EBEE`, 50 `#F6F7F8`, white `#FFFFFF`.
- Акцент (бирюза): `#12A594`, hover `#0E8576`, tint `#DCF2EF`, text-on-tint `#0B6B5F`.
- Статусы: Not started `#7C838D`, In progress `#12A594`, Completed `#6AA632` (тёплый зелёный, отстроен от бирюзы), Danger `#D14B4B`.
- Акцент применяется дозированно — только на действиях и активности (кнопки, прогресс, активная вкладка, play). Всё остальное графитовое.

**Типографика:** Inter, веса 400/500. H1 22/500 · H2 18/500 · H3 16/500 · Body 16/400 · Caption 13/400.

**Компоненты:** кнопки (Primary бирюза, Secondary обводка, Ghost, Disabled; высота ≥ 44px), статус-бейджи (цвет + точка-маркер — статус читается без опоры на цвет), карточки (белые на фоне `#F6F7F8`), прогресс-бары бирюзовые, чипы длительности на превью.

**Прочее:** иконки Tabler (outline), отступы и сетка кратно 4px, радиусы ~8–14px. В проде — лёгкие тени на карточках (в макетах глубину держат рамки и фон). Темы: светлая готова; тёмная (графит → почти-чёрный, бирюза ярче) — концепт описан, мокапы ещё не отрисованы.

**Доступность:** WCAG 2.1 AA, цвет не единственный носитель смысла, тач-цели ≥ 44px.

## 7. Адаптив / mobile-first

- Один продукт на двух размерах: боковое меню (десктоп) ↔ нижняя панель вкладок (мобайл, 4 вкладки у рабочего, +`Users` у привилегированных).
- Контент складывается в один столбец; ряды (Recommended, витрина категорий) листаются горизонтально свайпом.
- Видео: встроенный плеер 16:9 в портрете, полный экран в ландшафте.

---

## 8. Тех-стек и интеграции (вход в Этап 6)

- **Frontend:** React 18, TypeScript (строгий), Vite, CSS Modules. Feature-based структура, переиспользуемые компоненты, без дублирования логики. Фронт презентационный — бизнес-логика на бэке.
- **Backend:** Python, PostgreSQL, REST API.
- **Auth:** username + password (standalone, без SSO пока).
- **Видео:** Vimeo (хранение, стриминг, адаптивное качество) + Vimeo Player эмбедом. Academy хранит только метаданные урока, vimeo id, прогресс, права. Пользователь не выходит на Vimeo напрямую.
- **i18n:** язык по умолчанию — English, приложение i18n-ready (текст в ключах, запас по ширине, локале-форматы дат/чисел). Другие языки подключаются ключами позже.

## 9. Не в MVP (future)

AI-ассистент, обучающий аватар, квизы, сертификаты, закладки, избранное, заметки, learning paths, уведомления, поиск по транскрипту, офлайн-режим, скачиваемые вложения к урокам.

## 10. Открытые задачи / что доделать по дизайну

- Тёмная тема — мокапы.
- Экраны привилегированной роли: `Users` (список + форма «создать пользователя»).
- Витрина (публичная маркетинг-страница) — детальный дизайн.
- Lesson player на мобайле — в цвете.
- Страница категории `Lessons` с подкатегориями и шапкой.
- Детализация будущих ограничений для `Manager` / `Site Manager`.
