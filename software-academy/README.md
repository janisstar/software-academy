# Software Academy

Корпоративный учебный портал для пользователей Manufacturing Platform: короткие
задачные видеоуроки (3–5 минут). Цель — сократить онбординг, снизить число
обращений в поддержку и давать командам на производстве непрерывное обучение.

Это **standalone-приложение** со своей авторизацией и брендом. Видео хранятся
на Vimeo; Academy хранит только метаданные урока, Vimeo ID, прогресс и права.

---

## Технологический стек

| Слой       | Технологии                                                        |
|------------|-------------------------------------------------------------------|
| Frontend   | React 18, TypeScript, Vite, CSS Modules *(этап после бэкенда)*    |
| Backend    | Python, FastAPI, SQLAlchemy 2.0, Alembic, Pydantic v2             |
| База данных| PostgreSQL                                                        |
| Auth       | username + password, серверные сессии (httpOnly-cookie)           |
| Видео      | Vimeo (хранение/стриминг) + Vimeo Player (эмбед)                  |
| Инфра      | Docker + docker-compose                                           |

## Структура репозитория

```
software-academy/
├── README.md            ← этот файл
├── .gitignore
├── docs/                ← документация (начни отсюда)
│   ├── 01-project-overview.md   — что это и зачем, роли, права доступа
│   ├── 02-backend-guide.md      — инструкция бэкенд-разработчика (Python)
│   ├── 03-database-guide.md     — база данных, схема, миграции
│   ├── 04-server-and-docker.md  — запуск локально и в Docker
│   └── 05-roadmap.md            — пошаговый план разработки («ты здесь»)
└── backend/             ← Python-бэкенд (наполняется по шагам из roadmap)
    └── app/
        ├── core/        — конфиг, безопасность, подключение к БД
        ├── models/      — таблицы БД (SQLAlchemy)
        ├── schemas/     — формы запросов/ответов (Pydantic)
        ├── services/    — бизнес-логика
        └── api/v1/       — эндпоинты (роуты REST API)
```

## С чего начать

1. Прочитай **`docs/01-project-overview.md`** — общая картина продукта.
2. Открой **`docs/05-roadmap.md`** — там пошаговый план. В нём всегда отмечено,
   на каком шаге мы находимся.
3. Дальше — `docs/02-backend-guide.md`, `03-database-guide.md`,
   `04-server-and-docker.md` по мере необходимости.

> Документация написана так, чтобы по ней могли работать и человек, и ИИ-агент:
> весь контекст проекта собран в `docs/`.
